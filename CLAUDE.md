# 작업 원칙

- 외부 라이브러리(SDK, API 클라이언트 등)는 자체 인터페이스로 래핑한 뒤 사용. 직접 import하여 사용하지 않는다.
- 인터페이스는 내부 구현이 아닌 사용하는 쪽의 관점에서 설계한다. 메서드명과 시그니처만 보고 동작을 예측할 수 있어야 한다.
- 새 추상화(헬퍼, 유틸, 공통 모듈)는 3회 이상 반복이 확인된 후에만 도입한다. 중복 2회까지는 그대로 둔다.
- 모든 선언(모듈, 함수, 타입 등)은 소비되는 맥락에서 가장 가까운 위치에 둔다. 여러 곳에서 공유되기 전까지 상위 경로로 올리지 않는다.
- 버그를 숨기지 않을 것, 과감하게 error를 throw한다.

# 테스트

- mock/stub 대신 fake 구현을 선호한다.
- 테스트는 공개 인터페이스(입력 → 출력)만 검증한다. 내부 호출 순서나 구현 세부사항을 단언하지 않는다.
